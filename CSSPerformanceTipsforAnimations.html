CSS Animation Performance Tips

Why Performance Matters

Smooth animations improve user experience and prevent jank. Use GPU-accelerated properties for best performance.

Use Transform Instead of Position Changes

❌ Bad: Animating left/top properties

Triggers layout recalculation, causing performance issues.

Bad Performance
@keyframes moveBad { 0% { left: 0; top: 0; } 100% { left: 100px; top: 100px; } } .bad-performance { position: relative; animation: moveBad 2s infinite alternate; }

✅ Good: Using transform

Uses GPU acceleration, no layout recalculation needed.

Good Performance
@keyframes moveGood { 0% { transform: translate(0, 0); } 100% { transform: translate(100px, 100px); } } .good-performance { animation: moveGood 2s infinite alternate; }

Use Transform for Scaling

❌ Bad: Animating width/height

Triggers layout recalculation and repaint.

Bad Scale
@keyframes scaleBad { 0% { width: 100px; height: 100px; } 100% { width: 150px; height: 150px; } } .bad-scale { animation: scaleBad 2s infinite alternate; }

✅ Good: Using transform scale

GPU accelerated, no layout recalculation.

Good Scale
@keyframes scaleGood { 0% { transform: scale(1); } 100% { transform: scale(1.5); } } .good-scale { animation: scaleGood 2s infinite alternate; }

Use Transform for Rotation

❌ Bad: Using filter for rotation

Poor performance, not GPU accelerated.

Bad Rotate
@keyframes rotateBad { 0% { filter: progid:DXImageTransform.Microsoft.BasicImage(rotation=0); } 100% { filter: progid:DXImageTransform.Microsoft.BasicImage(rotation=1); } } .bad-rotate { animation: rotateBad 2s infinite linear; }

✅ Good: Using transform rotate

GPU accelerated, smooth rotation.

Good Rotate
@keyframes rotateGood { 0% { transform: rotate(0deg); } 100% { transform: rotate(360deg); } } .good-rotate { animation: rotateGood 2s infinite linear; }

GPU Acceleration Techniques

Using will-change

Will Change
.will-change { will-change: transform; animation: moveGood 2s infinite alternate; }

Tells the browser to optimize for changes to the transform property.

Using transform: translateZ(0)

GPU Accelerated
.gpu-accelerated { transform: translateZ(0); animation: moveGood 2s infinite alternate; }

Forces GPU acceleration by creating a new stacking context.

Using backface-visibility

Backface Hidden
.backface-visibility { backface-visibility: hidden; animation: rotateGood 2s infinite linear; }

Can help with performance in some browsers by optimizing 3D transforms.

Additional Performance Tips

✅ Use opacity for fade effects

Opacity is GPU accelerated and doesn't trigger layout.

.fade { transition: opacity 0.3s ease; } .fade:hover { opacity: 0.5; }

❌ Avoid animating box-shadow

Box-shadow triggers repaint and can be expensive.

✅ Alternative: Use pseudo-elements

Animate opacity of a pseudo-element with box-shadow instead.

.shadow-effect { position: relative; } .shadow-effect::after { content: ''; position: absolute; top: 0; left: 0; width: 100%; height: 100%; box-shadow: 0 10px 20px rgba(0,0,0,0.2); opacity: 0; transition: opacity 0.3s ease; } .shadow-effect:hover::after { opacity: 1; }
