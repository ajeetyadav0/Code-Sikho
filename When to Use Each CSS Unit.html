When to Use Each CSS Unit

Choose the Right Unit for the Right Job

Different CSS units serve different purposes. Understanding when to use each one is key to creating maintainable and responsive designs.

Quick Decision Guide

What are you styling?

Typography

Use rem for consistency

Use em for component-relative sizing

Use clamp() for responsive text

Layout & Spacing

Use rem for consistent spacing

Use % for fluid layouts

Use vh/vw for viewport-based sizing

Borders & Outlines

Use px for precision

Typically 1px, 2px, or 3px

Never use relative units

Components

Use rem for overall sizing

Use em for internal scaling

Use px for fine details

Typography: Use rem & clamp()

Main Heading with clamp()

Subheading with clamp()

Body text uses rem units for consistent sizing across the site.

Small text also uses rem units for accessibility.

/* Responsive Typography */ h1 { font-size: clamp(2.4rem, 6vw, 4rem); /* Min: 24px, Preferred: 6vw, Max: 40px */ } h2 { font-size: clamp(2rem, 5vw, 3.2rem); /* Min: 20px, Preferred: 5vw, Max: 32px */ } body { font-size: 1.6rem; /* 16px base size */ } .small-text { font-size: 1.4rem; /* 14px */ }

Components: Use em for Internal Scaling

/* Button Component */ .btn { padding: 1.2rem 2.4rem; /* rem for base size */ font-size: 1.6rem; /* rem for text */ border-radius: 0.4rem; /* rem for radius */ } .btn-small { padding: 0.8em 1.6em; /* em for relative scaling */ font-size: 0.875em; /* em for relative text */ } .btn-large { padding: 1.2em 2.4em; /* em for relative scaling */ font-size: 1.25em; /* em for relative text */ }

Use Case Summary

üìè Use px When:

  • Borders and outlines
  • Box shadows (blur radius)
  • Media query breakpoints
  • Minimum/maximum values in clamp()
  • Very small precise values

üìê Use rem When:

  • Font sizes for body text
  • Component dimensions
  • Spacing and margins
  • Border radius
  • Most sizing needs

üîÑ Use em When:

  • Component variations
  • Icon sizing
  • Elements that should scale with parent
  • Media queries within components
  • Relative padding/margins

  • Full-screen sections
  • Hero images

  • Fluid grid layouts

Modern CSS Unit Strategy

/* Modern CSS Unit Strategy */ :root { /* Base sizing */ font-size: 62.5%; /* Makes 1rem = 10px */ /* Spacing scale */ --space-xs: 0.4rem; /* 4px */ --space-sm: 0.8rem; /* 8px */ --space-md: 1.6rem; /* 16px */ --space-lg: 2.4rem; /* 24px */ --space-xl: 3.2rem; /* 32px */ /* Border scale */ --border-sm: 1px; --border-md: 2px; --border-lg: 4px; } /* Typography */ body { font-size: 1.6rem; /* 16px */ } h1 { font-size: clamp(2.4rem, 6vw, 4rem); } h2 { font-size: clamp(2rem, 5vw, 3.2rem); } /* Components */ .card { padding: var(--space-lg); border: var(--border-sm) solid #ddd; border-radius: 0.8rem; margin-bottom: var(--space-md); } /* Layout */ .container { max-width: 120rem; padding: 0 var(--space-md); margin: 0 auto; } .grid { display: grid; gap: var(--space-lg); } /* Full viewport sections */ .hero { height: 100vh; padding: var(--space-xl) var(--space-md); }
