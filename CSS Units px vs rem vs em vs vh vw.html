CSS Units Explained: px vs rem vs em vs vh/vw

Understanding CSS Units

CSS units determine the size of properties like width, height, font-size, and more. Choosing the right unit is crucial for responsive design.

Visual Comparison

px 200px × 100px
rem 12.5rem × 6.25rem
em 12.5em × 6.25em
vh/vw 25vw × 25vh

px (Pixels)

This text uses px units (16px)
This text uses px units (24px)
.px-example { font-size: 16px; width: 200px; height: 100px; }

✅ Fixed size, consistent across devices

✅ Precise control over dimensions

⚠️ Not scalable for accessibility

⚠️ Doesn't adapt to user preferences

rem (Root EM)

This text uses rem units (1rem = 16px)
This text uses rem units (1.5rem = 24px)
.rem-example { font-size: 1rem; /* 16px if root is 16px */ width: 12.5rem; /* 200px if root is 16px */ }

✅ Relative to root font size

✅ Scalable and accessible

✅ Consistent across components

rem is always relative to the :root (html) element's font-size, not parent elements.

em (Relative to Parent)

This text uses em units (1em = 16px)
This text uses em units (1.5em = 24px)
.em-example { font-size: 1em; /* Same as parent */ width: 12.5em; /* Relative to parent font-size */ }

✅ Relative to parent font size

✅ Good for scalable components

⚠️ Can cause compounding issues

⚠️ Harder to predict final size

em vs rem: The Compounding Problem

em Units (Compounding)

Parent: 20px
Child: 1.2em = 24px
Grandchild: 1.2em = 28.8px

rem Units (No Compounding)

Parent: 20px
Child: 1.2rem = 19.2px
Grandchild: 1.2rem = 19.2px

Notice how em units compound with each nesting level, while rem units stay consistent.

vh/vw (Viewport Units)

100vw width (full viewport width)
50vh height (half viewport height)
.viewport-example { width: 100vw; /* 100% of viewport width */ height: 50vh; /* 50% of viewport height */ font-size: 2vw; /* 2% of viewport width */ }

✅ Perfect for full-screen layouts

✅ Responsive to viewport size

✅ Great for typography scaling

⚠️ Can cause issues on mobile (viewport changes)

⚠️ May be too small/large on extreme screen sizes

Responsive Typography with clamp()

This text uses clamp(16px, 4vw, 24px) - it scales between 16px and 24px based on viewport width.
Resize your browser to see how the font size adapts!
.responsive-text { font-size: clamp(16px, 4vw, 24px); /* Minimum: 16px */ /* Preferred: 4% of viewport width */ /* Maximum: 24px */ }

✅ Sets min, preferred, and max values

✅ Perfect for fluid typography

✅ No media queries needed

Comparison Table

Unit Relative To Best For Pros Cons
px Absolute Borders, precise layouts Precise, predictable Not accessible
rem Root font-size Typography, spacing Scalable, consistent Requires calculation
em Parent font-size Components, icons Relative, scalable Can compound
vh/vw Viewport Full-screen layouts Responsive, fluid Mobile issues

Best Practices

✅ Recommended Approach:

  • Use rem for typography and spacing
  • Use px for borders (1px, 2px)
  • Use em for component-specific scaling
  • Use vh/vw for full-screen elements
  • Use clamp() for responsive typography
  • Set root font-size to 62.5% for easier rem calculations (1rem = 10px)
/* Modern CSS Units Best Practice */ :root { font-size: 62.5%; /* Makes 1rem = 10px */ } body { font-size: 1.6rem; /* 16px */ } h1 { font-size: clamp(2.4rem, 5vw, 3.6rem); /* 24px to 36px */ } .container { padding: 2rem; /* 20px */ border: 1px solid #ddd; /* Use px for borders */ } .component { font-size: 1.2em; /* Relative to parent */ margin: 1em 0; } .hero { height: 100vh; /* Full viewport height */ font-size: 4vw; /* Scales with viewport */ }
