CSS Unit Best Practices

Build Maintainable and Scalable CSS

Following CSS unit best practices ensures your code is consistent, accessible, and easy to maintain across projects.

Establish a Unit System

/* CSS Unit System with Variables */ :root { font-size: 62.5%; /* Makes 1rem = 10px */ /* Spacing Scale - Based on 8px grid */ --space-xs: 0.4rem; /* 4px */ --space-sm: 0.8rem; /* 8px */ --space-md: 1.6rem; /* 16px */ --space-lg: 2.4rem; /* 24px */ --space-xl: 3.2rem; /* 32px */ /* Typography Scale */ --text-xs: 1.2rem; /* 12px */ --text-sm: 1.4rem; /* 14px */ --text-base: 1.6rem; /* 16px */ --text-lg: 1.8rem; /* 18px */ --text-xl: 2.4rem; /* 24px */ /* Border Scale */ --radius-sm: 0.4rem; /* 4px */ --radius-md: 0.8rem; /* 8px */ --border-sm: 1px; --border-md: 2px; }

✅ Consistent scale across the project

✅ Easy to maintain and update

✅ Semantic naming conventions

Core Best Practices

1. Use rem for Most Sizing

rem should be your default unit for:

  • Font sizes
  • Component dimensions
  • Spacing and margins
  • Padding
  • Border radius

2. Use px Only for Borders

Always use px for:

  • Borders (1px, 2px)
  • Box shadows
  • Media query breakpoints
  • Minimum/maximum values

3. Use em for Component Scaling

em is perfect for:

  • Component variations
  • Icon sizing
  • Elements that scale with parent
  • Relative padding within components

4. Use vh/vw for Viewport Elements

Viewport units for:

  • Full-screen sections
  • Hero images
  • Responsive typography
  • Fluid layouts

Do's and Don'ts

✅ Do

  • Set a consistent base font size
  • Create a spacing scale with rem
  • Use semantic variable names
  • Round rem values to 2-3 decimals
  • Test with different font sizes
  • Document your unit system
  • Use clamp() for responsive typography

❌ Don't

  • Mix units randomly
  • Use rem for borders
  • Convert px to exact rem values
  • Nest em too deeply
  • Ignore accessibility
  • Use viewport units for everything
  • Hard-code values without variables

Best Practices Checklist

Project Setup

  • Set root font-size to 62.5% for easier rem calculations
  • Create CSS variables for spacing, typography, and borders
  • Establish consistent naming conventions
  • Document your unit system

Development

  • Use rem for most sizing needs
  • Use px only for borders and fine details
  • Use em for component-relative sizing
  • Use viewport units for full-screen elements

Accessibility

  • Test with different font sizes
  • Ensure text remains readable
  • Don't disable user zoom
  • Use relative units for scalability

Performance

  • Avoid overly precise decimal values
  • Use CSS variables for maintainability
  • Minimize unit conversions
  • Test across different devices

Common Pitfalls to Avoid

⚠️ The "Perfect Conversion" Trap

Don't waste time converting 23px to exactly 1.4375rem. Round to 1.4rem or 1.44rem. The visual difference is negligible, but the code readability improves significantly.

⚠️ Inconsistent Unit Usage

Avoid mixing units arbitrarily within the same property type. If you use rem for margins, use rem for all margins unless there's a specific reason not to.

⚠️ Ignoring Context

Remember that em units compound. A 1.2em element inside a 1.2em parent becomes 1.44em, not 1.2em. Use rem when you need consistency across nesting levels.

⚠️ Over-relying on Viewport Units

Don't use vw/vh for everything. They can cause issues on mobile devices where the viewport changes dynamically. Use them strategically for specific use cases.
